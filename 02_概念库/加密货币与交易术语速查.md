# 加密货币与交易术语速查

本页参考《加密货币名词解释.docx》整理，但不照搬原文。定义优先服务于 BTC 现货、永续合约和交易复盘，并把不可验证的“庄家叙事”改写为可观察的价格行为。

## 使用方法

- “定义”回答它客观上是什么。
- “系统中的用法”回答交易前后应记录什么。
- 黑话只能帮助理解交流，不能直接成为入场依据。
- 交易所的实际订单、保证金、资金费和强平规则，以所用平台当时的合约说明为准。

## 1. BTC、网络与资产保管

| 术语 | 操作化定义 | 系统中的用法 |
| --- | --- | --- |
| Bitcoin / BTC | 去中心化点对点电子现金系统及其原生资产；BTC 是常用交易代码。 | 本系统默认研究 `BTCUSDT`，但必须区分现货与永续合约。 |
| 区块链 | 按共识规则维护的交易记录数据结构。 | 不作为短线入场信号。 |
| 节点 | 验证、保存或转发网络数据的软件实例。 | 用于理解网络，不与交易所账户混为一谈。 |
| 钱包 | 管理密钥、生成收款地址并签名交易的软件或设备；币并非“存放在 App 里”。 | 记录是自托管还是托管于交易所。 |
| 私钥 | 控制相应链上资产支出权限的秘密数据。 | 不截图、不上传、不交给任何人。 |
| 助记词 / 恢复短语 | 可恢复钱包密钥的一组词。 | 离线备份；获得助记词等同于可能获得资产控制权。 |
| 冷钱包 | 私钥签名环境通常不直接联网的保管方式。 | 降低在线攻击面，但不消除备份、供应链和操作风险。 |
| 热钱包 | 日常连接网络的钱包。 | 便利性高，暴露面也更大；只保留必要余额。 |
| 交易所托管 | 平台代表用户控制资产或合约账户。 | 平台风险、提币限制和账户安全是独立风险项。 |
| 法币 | 由主权国家发行并规定为法定货币的货币。 | 与“稳定币”不同。 |
| 稳定币 | 设计目标是跟踪某种参考资产价格的加密资产。 | `USDT` 是报价资产，不等同于银行美元存款。 |
| 交易对 | 以基础资产/报价资产表示相对价格，如 `BTC/USDT`。 | 明确产品、计价币和手续费币种。 |

## 2. 市场、合约与账户

| 术语 | 操作化定义 | 系统中的用法 |
| --- | --- | --- |
| 现货 | 直接买卖标的资产并完成账户余额交割。 | 与合约交易分开统计。 |
| 期货 | 约定未来结算或交割规则的标准化衍生合约。 | 查看到期、结算和保证金规则。 |
| 永续合约 | 没有传统到期日、通常用资金费等机制锚定现货的衍生合约。 | 每笔交易记录 Funding、标记价格和强平规则。 |
| 保证金 | 为维持杠杆头寸而提供的履约担保资金。 | 不等同于该笔交易的最大可接受风险。 |
| 杠杆 | 用较少保证金控制更大名义头寸。 | 放大盈亏和强平风险，不提升策略胜率。 |
| 名义价值 | 头寸所代表的标的总价值。 | 仓位计算以名义价值、止损距离和最大风险共同约束。 |
| 全仓保证金 | 多个头寸共享账户可用保证金的模式。 | 一个头寸可能影响整个账户；初学阶段慎用。 |
| 逐仓保证金 | 为单个头寸分配相对独立保证金的模式。 | 限制保证金影响范围，但仍不能替代止损。 |
| 强平 | 账户风险达到平台阈值后，头寸被强制减仓或平仓。 | 计划止损必须在强平之前，且预留滑点。 |
| 资金费率 Funding | 永续合约多空持仓者之间按规则交换的费用，用于帮助合约价格贴近现货。 | 不把正 Funding 简化为“一定跌”，只记作持仓成本和拥挤度背景。 |
| 多头 / 做多 | 从价格上涨中获利的头寸。 | 必须有入场、失效点、目标和仓位。 |
| 空头 / 做空 | 从价格下跌中获利的头寸。 | 与做多使用同一风险标准。 |
| 开仓 / 平仓 | 建立头寸 / 结束全部或部分头寸。 | “建仓”不等于只买入；空头也有开仓。 |
| 加仓 / 减仓 | 增加 / 减少已有头寸。 | 只允许预先计划的加仓；禁止浮亏摊平。 |
| 对冲 | 用相关、方向相反的头寸降低某类风险敞口。 | 对冲不是“避免认错”，不能代替原头寸止损。 |

## 3. 订单、流动性与成交

| 术语 | 操作化定义 | 系统中的用法 |
| --- | --- | --- |
| 订单簿 | 按价格排列的未成交买卖委托集合。 | 观察盘口时仍以计划失效点为准。 |
| Bid / Ask | 当前最高买价 / 最低卖价。 | 市价买通常吃 Ask，市价卖通常吃 Bid。 |
| 点差 Spread | 最优 Ask 与最优 Bid 的差。 | 点差扩大时降低仓位或放弃交易。 |
| 市场深度 | 不同价位可成交委托量的分布。 | 深度薄时更容易产生滑点。 |
| 流动性 | 在较小价格冲击下完成交易的能力。 | SMC 的“流动性位置”是价格行为假设，不等同于真实订单簿全貌。 |
| 滑点 | 预期成交价与实际成交价的差异。 | 回测和复盘必须计入手续费与滑点。 |
| 市价单 | 以当前可获得的最佳价格尽快成交的订单。 | 优先保证成交，不保证某个精确价格。 |
| 限价单 | 只在指定价格或更优价格成交的订单。 | 控制最差成交价，但可能不成交。 |
| 止损市价单 | 触发后转为市价执行的止损订单。 | 极端行情可能滑点，但通常优先退出。 |
| 止损限价单 | 触发后提交限价单。 | 价格跳过限价时可能无法成交，不能误认为必然止损。 |
| Maker | 订单进入订单簿并提供流动性的一方。 | 费率可能更低，但不能为了省费牺牲风险退出。 |
| Taker | 订单立即与现有委托成交并消耗流动性的一方。 | 市价单通常是 Taker。 |
| Post-only | 保证订单只以 Maker 方式进入订单簿，否则拒单或取消。 | 适合计划限价入场，不适合紧急止损。 |
| 撤单 | 在未完全成交前取消剩余委托。 | 撤单成功前不能假定风险已消失。 |

## 4. 风险、绩效与执行

| 术语 | 操作化定义 | 系统中的用法 |
| --- | --- | --- |
| 仓位 | 当前头寸相对账户的风险或名义价值暴露。 | 不只看“用了多少保证金”，核心看止损触发时亏多少。 |
| 单笔风险 | 从计划入场到失效止损的预计最大损失。 | 初始研究阶段默认账户净值的 `0.25%–0.5%`。 |
| 止损 | 当交易假设失效时退出的价格或条件。 | 先定失效点，再算仓位；禁止事后放宽。 |
| 止盈 | 在预定目标或条件满足时退出盈利头寸。 | 优先对应目标流动性或结构目标。 |
| 盈亏比 RR | 计划潜在收益与计划潜在风险的比值。 | 低于系统最低 RR 时不交易。 |
| R 倍数 | 实际盈亏除以该笔初始风险。 | 用于跨仓位、跨价格比较交易质量。 |
| 已实现 / 未实现盈亏 | 已平仓形成的盈亏 / 当前持仓的浮动盈亏。 | 绩效统计只以实际规则为准，不用浮盈证明系统有效。 |
| 回撤 | 权益从阶段高点到后续低点的下降。 | 与胜率同时观察；优先控制最大回撤。 |
| 胜率 | 盈利交易数占已平仓有效交易数的比例。 | 必须与平均盈利、平均亏损和样本量一起看。 |
| Profit Factor | 总盈利绝对值除以总亏损绝对值。 | 小样本下不做强结论。 |
| 期望值 | 单笔交易长期平均预期结果。 | `胜率 × 平均盈利R + 败率 × 平均亏损R`。 |

## 5. 行情术语与常见黑话

| 术语 | 更严谨的理解 | 不应如何使用 |
| --- | --- | --- |
| 牛市 / 熊市 | 较长时间尺度上的上升 / 下降市场环境。 | 不因标签而忽略当前结构失效。 |
| 趋势 | 一系列可定义的高低点或统计方向。 | 不能只凭主观斜率判断。 |
| 震荡 / 盘整 / 区间 | 价格在可识别边界内反复交易。 | “横盘久了必涨/必跌”没有方向信息。 |
| 突破 / 跌破 | 价格越过结构边界；是否有效需定义收盘、位移或回踩条件。 | 单根影线越界不自动等于有效突破。 |
| 回踩 / 回抽 | 突破后重新测试原边界或失衡区域。 | 碰到线不代表一定反转。 |
| 插针 | 短时间快速越界后收回，留下长影线。 | 可作为 Sweep 候选，不是单独入场信号。 |
| 超买 / 超卖 | 指标或分布处于预设极端区间。 | 不等于价格“即将必跌 / 必涨”。 |
| 反弹 / 回调 | 主方向中的短暂反向运动，需先定义参考周期。 | 不预设一定延续或反转。 |
| 主升浪 / 主跌浪 | 对显著趋势段的事后或主观描述。 | 不作为可验证入场条件。 |
| 诱多 / 诱空 | 交易者对假突破、流动性扫取或失败延续的俗称。 | 不假定能知道某个“主力”的意图。 |
| 拉盘 / 砸盘 / 吸筹 / 庄家 | 市场黑话，常把价格变化归因于不可见主体意图。 | 复盘时改写成成交、波动、结构与流动性事实。 |
| 瀑布 / 跳水 | 短时间快速大幅下跌。 | 不用情绪词替代波动率和结构记录。 |
| 套牢 / 解套 / 割肉 / 踏空 | 对持仓亏损、回本退出、亏损退出、错过行情的情绪化描述。 | 不因成本价、后悔或面子改变失效规则。 |
| 搬砖 | 利用不同场所价差进行套利。 | 必须计入转账时间、手续费、限额和对手方风险。 |
| 大饼 / 币王 | BTC 的中文社区俗称。 | 仅用于交流，不用于记录字段。 |
