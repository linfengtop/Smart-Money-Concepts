# BTC SMC 学习与交易系统

这是一个围绕 BTC“聪明钱概念”（SMC/ICT）建立的学习、执行、回测与复盘工作区。

## 核心原则

SMC 是描述价格行为的一套语言，不是天然有效的完整策略。本系统只保留能够被客观定义、重复识别和统计验证的部分：

`高周期环境 → 流动性位置 → 扫流动性 → 位移 → MSS → 回踩 → 风险受控的执行`

任何单独出现的 FVG、Order Block 或 BOS 都不是入场理由。

## 文件夹导航

- `01_交易系统/`：当前实际执行的规则、入场模型和检查清单。
- `02_概念库/`：市场结构、流动性、FVG 等概念的操作化定义。
- `03_学习计划/`：八周学习与验证路线。
- `04_复盘/`：每日、每周复盘方法和错误分类。
- `05_资料/`：权威资料与研究边界。
- `outputs/btc-smc-system/`：正式版交易日志工作簿。
- `BTC合约交易记录-简化版.xlsx`：原始记录，保留不动。

## 每天怎么用

1. 打开工作簿的“参数设置”，确认账户和风险参数。
2. 在“盘前计划”记录高周期结构、两侧流动性和当日禁做条件。
3. 只有满足《交易系统规则 v0.1》的 A 级模型才允许执行。
4. 交易结束后在“交易日志”填写完整数据和违规标签。
5. 每周查看“统计看板”，重点观察期望值、Profit Factor、最大回撤和按计划执行率。

## 当前研究假设

当 BTC 在高周期背景一致的前提下，扫过明确外部流动性，并以位移形成低周期 MSS，随后首次回踩该位移产生的 FVG 时，可能存在正期望入场机会。

这只是待验证假设。达到至少 100 个合格样本前，不应把短期盈利视为系统已经有效。
