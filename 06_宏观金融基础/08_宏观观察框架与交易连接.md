# 宏观观察框架与交易连接

目标不是每天解释所有新闻，而是维护一个小型、可回溯的状态面板。

## 1. 四层决策栈

1. **宏观环境**：增长、通胀、政策和信用。
2. **跨资产确认**：2Y、10Y、实际利率、DXY、黄金、美股、原油和信用利差。
3. **BTC 市场状态**：现货资金流、杠杆、波动和高周期结构。
4. **执行**：SMC/常规价格结构、入场、止损、仓位和失效。

宏观只调整方向偏好、持仓预期和风险预算，不允许跳过第 3、4 层。

## 2. 最小宏观面板

| 模块 | 指标 | 问题 |
|---|---|---|
| 增长 | GDP、PMI/ISM、新订单、零售 | 增长在加速还是减速？ |
| 就业 | 非农、失业率、参与率、时薪、初请 | 劳动力市场是否再平衡？ |
| 通胀 | CPI、核心 CPI、PCE、核心 PCE、油价 | 通胀方向、广度和粘性？ |
| 政策 | FOMC、市场隐含路径、Fed 资产负债表 | 相对预期更鹰还是更鸽？ |
| 利率 | 2Y、10Y、10Y 实际利率、曲线 | 是政策、增长还是期限溢价？ |
| 风险 | 信用利差、VIX、银行压力 | 融资和风险承受是否恶化？ |
| 美元 | DXY、广义美元、主要利差 | 全球美元条件松还是紧？ |
| 商品 | 黄金、WTI/Brent、库存 | 避险、通胀还是需求主导？ |
| BTC | ETF/现货流、资金费率、OI、结构 | 宏观是否被内部资金确认？ |

## 4. 情景而不是点预测

至少维护三个情景：

- **基准情景**：当前最可能。
- **上行风险**：增长/通胀或风险资产比预期强。
- **下行风险**：衰退、信用或流动性冲击。

概率总和为 100%，但重点不在伪精确，而在提前写出触发条件。

示例：

| 情景 | 条件 | 跨资产确认 | BTC 处理 |
|---|---|---|---|
| 软着陆 | 通胀降、增长稳 | 实际利率温和下、利差稳 | 回调优先找多，但等结构 |
| 再通胀 | 通胀与增长再升 | 2Y、DXY、实际利率上 | 降低多头风险，关注高位失败 |
| 衰退 | 就业和信用恶化 | 收益率下但利差/VIX 上 | 先防去杠杆，不抢降息利多 |
| 流动性缓解 | 融资压力下降 | 美元弱、利差收、现货流入 | 宏观顺风，提高 A 级机会权重 |

## 5. 数据事件如何连接到 SMC

### 事件前

- 标注前高、前低、日/周流动性。
- 写出高于、符合、低于预期三种情景。
- 明确不在数据前新开高杠杆仓位。

### 事件后

1. 不追第一段冲击。
2. 看收益率和 DXY 是否确认。
3. 看 BTC 是否扫过外部流动性。
4. 等位移、MSS/CHOCH 和首次有效回踩。
5. 止损放在结构失效处，不因宏观观点移动。

### 方向冲突

- 宏观偏多、结构偏空：不做多；等待结构重新转多。
- 宏观偏空、结构偏多：可以不做，或按更低风险预算处理已验证模型。
- 宏观与结构一致：只是提高候选质量，仍不保证盈利。

## 6. 风险预算调整

宏观不应用来扩大单笔最大亏损。可以调整的是：

- 是否参与；
- 只做一个方向还是双向；
- 持仓时间预期；
- 分批止盈速度；
- 事件前是否减仓；
- 同方向相关仓位的总风险。

例如 BTC、多只科技股和高贝塔山寨币可能是同一个“流动性风险”暴露，不能当作分散。

## 7. 判断是否真正理解一条叙事

一条可交易的宏观叙事必须能回答：

1. 新信息是什么？
2. 与市场预期差多少？
3. 通过哪条资产负债表和价格链传导？
4. 哪个市场应该最先确认？
5. 多久可能生效？
6. 什么证据会证伪？
7. 当前价格是否已经反映？

答不出时，它只是故事，不是决策依据。

## 8. 推荐节奏

- 每日 5–10 分钟：收益率、DXY、黄金、原油、美股、BTC 和事件日历。
- 每周 30–45 分钟：更新状态、情景、关键位和风险预算。
- 每月 60 分钟：读取 CPI/PCE、就业、PMI、消费和政策变化。
- 每季度：复核 GDP、盈利、信贷和长期假设。

少而稳定的面板比不断添加指标更有效。
